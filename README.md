# CNX PBB MASTER – Web-based Prototype

## System Overview
ระบบนี้เป็น web-based prototype เพื่อสนับสนุนงานวิจัยด้านการใช้สะพานเทียบเครื่องบิน  
(Passenger Boarding Bridge: PBB) โดยใช้เป็นเครื่องมือในการถ่ายทอดประสบการณ์ของผู้เชี่ยวชาญ  
ให้กับบุคลากรรุ่นใหม่ เพื่อลดความผิดพลาดในการปฏิบัติงานและเพิ่มความเข้าใจเชิงสถานการณ์

ระบบนี้ถูกพัฒนาเป็นต้นแบบเพื่อใช้ประกอบการเก็บข้อมูลจากแบบสอบถามและการสัมภาษณ์  
และใช้เป็นสื่อประกอบการวิเคราะห์ในงานวิจัย

---

## System Architecture
ระบบทำงานในรูปแบบ **Front-end only** โดยรันบน **Web Browser**  
พัฒนาโดยใช้เทคโนโลยีดังนี้
- HTML – โครงสร้างหน้าเว็บ
- CSS – การจัดรูปแบบและการแสดงผล
- JavaScript – ตรรกะการทำงานของระบบ

ปัจจุบันระบบยังไม่มีการเชื่อมต่อฐานข้อมูลหรือ server  
ข้อมูลทั้งหมดถูกกำหนดไว้ในระบบในรูปแบบ Master Data เพื่อใช้เป็นต้นแบบ

---

## Input – Process – Output

### Input
- การเลือกหลุมจอดอากาศยาน
- การเลือกแบบอากาศยาน

### Process
- ตรวจสอบเงื่อนไขความเหมาะสมของอากาศยานกับหลุมจอด (เช่น จำนวนเครื่องยนต์)
- เรียกข้อมูลทางเทคนิคและคำแนะนำที่กำหนดไว้ในระบบ
- จัดลำดับและแสดงข้อมูลตามโครงสร้างที่ออกแบบไว้

### Output
- ข้อมูลทางเทคนิคของอากาศยาน
- คำเตือน ข้อห้าม และคำแนะนำในการปฏิบัติงาน
- การแสดงผลเชิงภาพผ่านหน้าเว็บ เพื่อช่วยการเรียนรู้เชิงสถานการณ์

---

## app.py
ไฟล์ `app.py` เป็นไฟล์โครงร่างสำหรับการพัฒนาระบบฝั่ง backend ในอนาคต  
เช่น การเชื่อมต่อฐานข้อมูล หรือการจัดการข้อมูลผู้ใช้งาน

ในเวอร์ชันปัจจุบัน ระบบยังทำงานแบบ front-end only  
และยังไม่มีการเรียกใช้งานไฟล์ `app.py`

---

## System Limitations
- ระบบยังเป็นต้นแบบ (Prototype) เพื่อการศึกษาและงานวิจัย
- ยังไม่มีการเชื่อมต่อฐานข้อมูลจริง
- ข้อมูลประสบการณ์ของผู้เชี่ยวชาญต้องได้จากการเก็บแบบสอบถามและการสัมภาษณ์
- ระบบยังไม่รองรับการใช้งานจริงในสภาพแวดล้อมสนามบิน
